# 喻鸣绘卷 Design.md

> 中文名：喻鸣绘卷
> 英文名：Yuming Scroll
> 副标题：AI 漫剧联动创作平台
> 用途：本文件是前端设计约束，给开发者和 AI 编码助手使用。以后新增页面、重构组件、调整样式时，应优先遵守这里的规则。

---

## 1. 产品气质

喻鸣绘卷不是单点生成器，而是一个“从故事到分镜、模型图、Flow Map、视频片段”的 AI 漫剧生产工作台。

整体气质应是：

- 专业创作工具，而不是营销落地页。
- 暗色工作台为主，突出沉浸、专注、可持续创作。
- 有国风绘卷与漫剧感，但不能做成古风装饰站。
- 有 AI 智能感，但不使用廉价科幻蓝光和堆满渐变的视觉。
- 核心体验是“资产可复用、节点可拖动、关系可追溯、结果可确认”。

一句话设计方向：

> 像一个给漫剧导演使用的 AI 制片台：左侧是创作资产，中央是可拖拽联动画布，弹窗里完成确认、生成和复用。

---

## 2. 设计原则

### 2.1 工具优先

不要把首页或工作台做成宣传页。首屏应直接让用户开始创作、进入项目或查看当前流程。

页面内容按真实工作流组织：

1. 输入故事或导入小说。
2. 生成并编辑文本、世界观、剧情大纲、分镜。
3. 生成人物模型图、场景模型图和风格参考图。
4. 在 Flow Map 中拖动节点、建立连接、复用资产。
5. 生成视频片段并进入 Gallery。

### 2.2 可追溯

任何生成结果都要让用户知道来自哪里：

- 人物模型、场景模型、风格图、脚本应有明确编号。
- 视频节点应显示连接输入：人物、场景、风格词、脚本。
- 已确认、生成中、失败、过期、待更新都要有可见状态。
- 连接关系直接影响生成结果，不能隐藏在二级菜单里。

### 2.3 可复用

人物模型和场景模型是平台优势，不是一次性素材。

前端必须突出：

- 人物模型、场景模型等资产节点可拖动。
- 一个资产可以重复连接到多个视频片段。
- 已连接关系要用明显颜色和锚点反馈。
- 断开连接、重新连接、重新生成后的状态变化要可见。

### 2.4 确认后再生成

漫剧生产中，生成前要让用户确认关键资产。

重要动作建议采用：

- 候选图选择：一次 3 张。
- 确认当前图：明确按钮和 confirmed 状态。
- 查看大图：独立预览。
- 保存到本地：稳定下载入口。
- 生成视频：只有关键输入明确后再推动。

## 3. 品牌元素

### 3.1 名称使用

正式中文名优先使用：

- 喻鸣绘卷

英文名作为辅助使用：

- Yuming Scroll

副标题：

- AI 漫剧联动创作平台

界面中不建议频繁重复英文名。中文名应出现在品牌、标题、封面、PPT 和对外页面中；工作台内部可以使用更短的功能名。

### 3.2 Logo

Logo 以“卷轴 + 漫画分格 + 电路 / AI 节点”为核心识别。

使用规则：

- 页面顶部、侧边栏、空状态、导出物中优先使用卷轴图标版。
- 不要把完整横版 logo 塞进小按钮或拥挤导航。
- Logo 周围保留足够留白，不能贴边、压字、叠在复杂背景上。
- 深色背景上使用白底 logo 时，外框要克制，避免像广告贴片。

### 3.3 寓意表达

不要在产品界面里大段解释“喻 / 鸣 / 绘卷”的含义。品牌寓意适合用于官网介绍、PPT 或对外材料，不适合占据工作台界面。

可用短句：

- 让故事被看见。
- 让角色、场景和分镜在同一条链路中联动。
- 从文本到片段，每一步都可确认。

## 4. 色彩系统

当前产品使用暗色工作台，主色来自深紫、墨黑、青绿和金色。色彩要保持克制，避免整页只有紫色渐变。

### 4.1 基础色

```css
:root {
  --bg: #0c0414;
  --surface: rgba(28, 21, 40, 0.88);
  --surface-strong: rgba(38, 30, 54, 0.95);
  --ink: #f8f3ff;
  --muted: rgba(255, 255, 255, 0.62);
  --line: rgba(255, 255, 255, 0.12);
}
```

### 4.2 品牌强调色

```css
:root {
  --accent: #a855f7;
  --accent-soft: #d8b4fe;
  --teal: #14b8a6;
  --cyan: #22d3ee;
  --gold: #d19a2a;
  --ok: #34d399;
  --warn: #fbbf24;
  --danger: #fb7185;
}
```

### 4.3 使用规则

- 紫色：主操作、AI 生成、当前选中、角色相关。
- 青色 / 蓝色：场景、风格图、系统连接、图像生成。
- 金色：品牌、标题点缀、关键商业信息，不要大量铺满。
- 绿色：ready、confirmed、完成、连接有效。
- 黄色：warning、需要确认、等待输入。
- 红色：失败、危险、删除、过期。

不要使用：

- 大面积纯紫渐变背景。
- 大面积米色 / 棕色复古界面。
- 低对比的灰字压在深紫背景上。
- 过多发光边框导致工具界面变成游戏 UI。

## 5. 字体与文字层级

### 5.1 字体

优先使用系统中文字体：

```css
font-family: "Microsoft YaHei", "PingFang SC", system-ui, sans-serif;
```

如果后续做官网或品牌页，可单独引入更有标题感的中文字体，但工作台内不要牺牲可读性。

### 5.2 层级

工作台建议：

- 页面 H1：32-44px。
- 区块标题：20-28px。
- 卡片标题：15-18px。
- 正文：13-15px。
- 状态 / 标签：11-13px。

不要在紧凑工具面板里使用巨大的标题。标题越大，文字越少。

### 5.3 文案风格

文案要像工具，不像宣传口号。

推荐：

- 生成 3 张候选图
- 确认这张
- 查看大图
- 存到本地
- 连接到人物
- 已确认
- 正在生成

避免：

- 开启你的无限想象
- 一键颠覆创作流程
- 赋能全链路智能生态
- 黑科技降临

## 6. 布局系统

### 6.1 工作台骨架

主工作台采用：

- 左侧窄导航，hover 或展开时显示文字。
- 中央为当前任务区。
- 复杂工作流页面使用大画布，不套在普通卡片里。
- 弹窗用于编辑、确认、候选图选择和视频预览。

### 6.2 尺寸节奏

建议间距：

- 页面外边距：24-32px。
- 主要区块间距：24px。
- 卡片内部：16-20px。
- 表单行间距：12-16px。
- 小标签间距：8px。

圆角：

- 工具按钮：8-12px。
- 输入框 / 小卡片：12px。
- 弹窗：18-24px。
- 图片候选卡：12-16px。

不要把所有元素都做成大圆角胶囊。胶囊只用于标签、状态和少量强按钮。

### 6.3 卡片使用

卡片只用于承载真实对象：

- 项目卡。
- 视频卡。
- 人物模型卡。
- 场景模型卡。
- 候选图卡。
- 表单输入块。

不要用“标题 + 三张空卡片”的通用咨询风格组织页面。

## 7. 页面规则

### 7.1 首页

首页任务是快速进入创作。

必须包含：

- 品牌入口。
- 一个清晰的创作输入框。
- 当前项目入口。
- 可点击示例题材。
- 加载 / 失败状态。

首页可以有氛围背景，但不能影响输入框可读性。

### 7.2 文本创作页

文本创作页要突出“两种入口，统一输出”：

- 无现成小说：故事灵感、世界观、剧情大纲、分镜脚本修改。
- 已有小说：上传或粘贴文本。
- 输出：可编辑、可保存、可继续修改的分镜脚本。

点击小框体打开弹窗输入，避免在主页面堆满大段文本。

### 7.3 人物模型页

人物模型页要强调角色一致性。

必须包含：

- 角色名称。
- 角色描述。
- 一致性 Prompt。
- 生成模型和画幅设置。
- 3 张候选图。
- confirmed 状态。
- 查看大图和存到本地。

人物模型图推荐三视图：正面、侧面、背面在同一张图中。

### 7.4 场景模型页

场景模型页要强调空间、氛围和后续复用。

必须包含：

- 场景名称。
- 场景描述。
- 场景 Prompt。
- 关键词标签。
- 3 张候选图。
- confirmed 状态。

场景图原则上避免人物、可读文字、logo、水印和真人照片。

### 7.5 Flow Map

Flow Map 是产品核心，不只是流程图。

必须突出：

- 节点可拖动。
- 画布可横向 / 纵向移动。
- 人物、场景、风格图、脚本可以重复连接到多个视频片段。
- 连接线颜色区分不同输入类型。
- 锚点状态可视化。
- 连接错误要即时反馈。
- 视频节点生成后可点击打开预览弹窗。

推荐结构：

- 左侧：可复用资产节点，人物和场景纵向排列。
- 中部：每段视频自己的脚本和风格输入。
- 右侧：15 秒视频生成节点和预览 / 输出。

连接颜色建议：

- 人物：紫色。
- 场景：青色。
- 风格图：蓝色。
- 脚本：金色或浅紫。

### 7.6 Gallery

Gallery 只展示已经生成并回填到项目资产里的视频。

必须包含：

- 视频缩略预览。
- 第几段 / 片段类型。
- 画幅比例。
- Provider。
- ready 状态。
- 播放弹窗。
- 存到本地。

不要在 Gallery 里重复展示文本创作内容。

## 8. 组件规则

### 8.1 按钮

主按钮：

- 用于生成、保存、确认、进入下一步。
- 可使用紫色或蓝色渐变。
- 文案必须是明确动作。

次按钮：

- 用于查看、取消、返回、下载。
- 使用透明或弱背景。

危险按钮：

- 用于删除、断开、清除。
- 使用红色或红色描边，必须有确认或足够清晰的文案。

图标：

- 使用 `lucide-react`。
- 工具按钮优先使用图标 + tooltip 或图标 + 短文字。
- 不要手写 SVG 图标，除非没有合适图标。

### 8.2 输入框

输入框必须：

- 有明确 label。
- 有 placeholder 示例。
- 有保存 / 取消动作。
- 长文本使用 textarea。
- 重要输入建议放弹窗，避免主页面过长。

### 8.3 弹窗

弹窗用于集中处理一个明确任务。

推荐结构：

- 顶部：类型标签、标题、状态、关闭按钮。
- 左侧：Prompt / 描述 / 参数。
- 右侧：候选图、预览或确认区。
- 底部：取消、保存、生成、确认。

弹窗背景可使用暗色遮罩和轻微 blur，但不要让主体内容过糊导致廉价感。

### 8.4 状态

必须统一使用这些状态词：

- idle：未开始。
- ready：可用 / 已完成。
- generating：生成中。
- confirmed：已确认。
- stale：已过期或需重新生成。
- failed：失败。

状态表达：

- ready / confirmed：绿色。
- generating：紫色或蓝色，配进度条。
- stale：黄色。
- failed：红色。

### 8.5 表格和数据

商业数据、成本对比、验证流程可以用表格或紧凑数据块。

规则：

- 数据要可扫描。
- 不要直接贴截图式表格。
- 降本、周期、人力这类核心数字可以放大。
- 小字说明要留足对比度。

## 9. 响应式规则

### 9.1 桌面端

桌面端是主要生产场景。

优先保证：

- Flow Map 大画布足够宽。
- 左侧导航不遮挡内容。
- 弹窗在 1366px 宽度下不溢出。
- 候选图区域至少能显示 3 个方案。

### 9.2 笔记本小屏

在 1280px 以下：

- 减少页面外边距。
- 面板可以从双栏变单栏。
- Flow Map 保持画布滚动，不要强行缩小到不可读。

### 9.3 手机端

手机端优先用于查看、确认和预览，不作为主要 Flow Map 编辑场景。

规则：

- 首页和 Gallery 必须可用。
- 文本输入弹窗必须可用。
- 图片候选确认必须可用。
- Flow Map 可以降级为横向滚动画布或分段列表视图。
- 不要让标题、按钮和卡片互相压住。

---

## 10. 动效规则

动效应该服务状态反馈，而不是装饰。

推荐：

- hover：轻微变亮或边框增强。
- loading：旋转图标、进度条、扫描线。
- 连接拖拽：线条实时跟随。
- confirmed：边框变绿、勾选图标。
- 弹窗：150-220ms 淡入 / 上移。

避免：

- 大面积持续漂浮光球。
- 页面元素无意义晃动。
- 过多粒子、星光、炫光。
- 影响输入和拖拽性能的动画。

---

## 11. 可访问性与可用性

必须做到：

- 按钮有 `aria-label` 或清晰文字。
- 弹窗可用 Escape 关闭。
- 重要操作有 disabled 状态。
- 文字不能被背景或图片遮挡。
- 深色背景上的正文对比度足够。
- 生成失败时给出清晰错误信息。
- 长任务给用户进度感。

不要让用户猜：

- 当前是否已保存。
- 哪张图已确认。
- 哪些节点已经连接。
- 为什么不能生成。
- 生成结果保存在哪里。

## 12. 前端实现约束

当前技术栈：

- Next.js
- React
- TypeScript
- Tailwind CSS
- 自定义 CSS
- lucide-react
- Radix UI 可用于弹窗、选择器、tooltip 等交互基础设施

实现规则：

- 优先复用现有组件和 CSS 变量。
- 不要引入新的 UI 框架，除非明确需要。
- 不要把可交互工作台做成静态图片。
- 不要用图片代替真实按钮、输入框和状态。
- 复杂生成面板可以拆组件，但不要过度抽象。
- 新增页面必须检查桌面和手机断点。

## 13. AI 编码助手执行规则

当用户要求“按 DESIGN.md 改前端”时，必须：

1. 先阅读本文件。
2. 检查当前相关组件和 CSS。
3. 保持暗色创作台、Flow Map、弹窗和候选图选择的产品语言一致。
4. 如果新增 UI，优先使用现有颜色变量、按钮风格、卡片风格和状态词。
5. 修改后用浏览器或截图检查布局，重点看文字是否被遮挡、是否溢出、手机端是否混乱。
6. 最终回复说明改了哪些页面、检查了什么。

如果设计目标不明确，优先选择：

- 更像专业工具，而不是宣传页。
- 更清楚的状态，而不是更炫的装饰。
- 更少但更准确的文案。
- 更强的资产复用和连接关系可见性。

## 14. 参考方式

本文件的格式参考了 `awesome-design-md` 这类“给 AI 编码助手读取的设计约束文件”思路，但内容应服务喻鸣绘卷自身品牌，不复制任何第三方产品的品牌视觉。

可借鉴的方向：

- AI 创作工具的沉浸工作台。
- 节点画布工具的拖拽、连接和状态可视化。
- 专业生产工具的低噪声信息密度。
- 国风绘卷的识别元素，但只作为品牌点缀。
