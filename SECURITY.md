# 安全政策

## 支持范围

当前项目处于早期开发阶段，只对 `main` 分支的最新版本提供安全修复。旧提交、个人 fork 和第三方部署由各自运营者负责维护。

## 报告漏洞

请不要通过公开 Issue 报告未修复漏洞、有效凭证、用户数据或可直接利用的细节。

优先使用 GitHub 仓库 Security 页面中的 Private vulnerability reporting。若该入口尚未启用，请通过维护者 GitHub 主页请求一个私密联系方式，并只发送最小必要信息。

报告建议包含：

- 受影响版本或提交
- 复现条件和最小复现步骤
- 可能影响的数据与权限范围
- 已验证的缓解方式（如有）

维护者会确认收到报告、评估影响，并在修复可用后协调披露时间。请给维护者合理的修复窗口。

## 部署者责任

公开源代码不等于安全的默认生产部署。运营者必须自行管理密钥、HTTPS、Cookie、数据库、对象存储、备份、日志、邮件、模型内容政策、访问控制和数据删除请求。

任何已经进入公开 Git 历史的密钥都应视为泄露并立即轮换；仅删除当前文件不足以恢复安全性。
